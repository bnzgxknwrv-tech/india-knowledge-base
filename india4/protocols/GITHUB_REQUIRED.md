# GITHUB REQUIRED

Vóór ieder inhoudelijk werk:
1. voer een echte repository-read uit;
2. voer een echte niet-destructieve write uit en verwijder of overschrijf die test in dezelfde opdracht;
3. lees de test terug.

Ontbreekt read of write, antwoord exact: `MARK: IK MIS GITHUB CONNECTOR!`

Een volledige rol blokkeert alleen bij:
- ontbrekende GitHub-read;
- ontbrekende GitHub-write;
- ontbrekende oorspronkelijke runopdracht;
- ontbrekende verplichte voorgangeroutput;
- aantoonbaar lege, afgekapt of corrupte voorgangeroutput;
- geldige bestaande completioncommit voor dezelfde rol;
- direct conflict met een expliciet Mark-besluit.

Niet blokkerend: één onzekere kandidaat, één ontbrekende bron, één onbekende locatie, geen directe visuele Google Maps-interface, ontbrekende protocol-SHA of technisch niet-kritieke metadata-afwijking.

Iedere rol schrijft, commit en leest zijn eigen bestanden terug. Geen hash-pinning of controllertransition vereist.
