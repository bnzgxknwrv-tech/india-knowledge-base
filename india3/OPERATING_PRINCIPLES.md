# INDIA3 OPERATING PRINCIPLES

1. GitHub is the only structural source of truth.
2. A chat is temporary and disposable.
3. Read the minimum pinned context required for the active role.
4. Validate the predecessor; never inherit claims, coordinates, statuses or source IDs blindly.
5. State exactly what was checked, how, by whom, when and with what uncertainty.
6. Run a tool-capability check before accepting external or GEO work.
7. Protect all active Mark decisions and explicit rejections.
8. Do not assign A/B/C for Mark; use `DOOR_MARK_TE_BEOORDELEN`.
9. Isolate candidate-level uncertainty instead of blocking unrelated valid work.
10. Complete as much as safely possible per activation; stop only on defined blockers.
11. Separate technical audit artifacts from practical user delivery.
12. Write recovery state before reporting failure.
