# LESSONS FROM INDIA 1 AND INDIA 2

1. `SYNTAX_VALID` is never equivalent to `CONTENT_VALID`.
Test: every validation statement names object, method, role, timestamp and residual uncertainty.

2. Capability acceptance precedes execution.
Test: every external or GEO task records `EXECUTABLE_DIRECT`, `EXECUTABLE_PUBLIC_SEARCH`, `REQUIRES_MARK_LINK`, or `NOT_EXECUTABLE` before research begins.

3. User delivery is a quality gate, not an optional post-process.
Test: candidate GOUD cannot pass without every required user-facing format existing and reopening successfully.

4. Mark's choices are protected data.
Test: worker output cannot replace, delete or reinterpret an active Mark decision; unchosen items remain `DOOR_MARK_TE_BEOORDELEN`.

5. GitHub carries context; chats execute temporary work.
Test: a role starts from ENTRYPOINT, NEXT_ACTION, one pinned manifest, required files and its contract only.

6. One activation should complete one role or one explicitly authorized multi-phase migration.
Test: technical substeps and commits occur internally without requiring Mark to paste predecessor content.

7. Context is minimized by reference, not by deleting evidence.
Test: historical outputs stay available, while manifests pin compact required subsets.

8. External research prompts must be executable contracts.
Test: each template contains role, object, scope, exclusions, source rules, uncertainty rules, output schema, capability check and stop conditions.

9. Successor roles verify rather than blindly inherit.
Test: every material retained claim records predecessor status and current verification action.

10. Exact communication prevents false assurance.
Test: broad labels such as checked, verified, definitive, complete or correct are forbidden without a qualified validation statement.

11. GEO evidence has levels.
Test: each coordinate records point type, evidence class, source, method, actual tool access, rejected alternatives and remaining uncertainty.

12. One disputed location does not poison an otherwise usable dataset.
Test: candidate-level blocking is isolated unless the disputed point is structurally required for all outputs.

13. A rejected coordinate is a durable constraint.
Test: inherited or official coordinates cannot restore it without a later explicit Mark decision.

14. Technical and user products remain separate but synchronized.
Test: IDs, counts, statuses and links match across audit registers, PDF/KML/CSV and delivery manifest.

15. Recovery is written before failure is reported.
Test: status, last valid commit, open step, required files and exact resume prompt exist in GitHub.
