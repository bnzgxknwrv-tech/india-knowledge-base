# USER COMMUNICATION

Use short operational responses. Do not paste technical reports into chat when direct artifacts are required.

Every validation statement must include the validated object and method. Preferred pattern:
`<object>: <status>; checked by <role> using <method/source> at <timestamp>; residual uncertainty: <text|none>.`

Forbidden unqualified terms: checked, verified, definitive, complete, correct.

A stop response names the blocker, committed status path and exact resume prompt. Missing GitHub read or write remains the sole exception and returns exactly `MARK: IK MIS GITHUB CONNECTOR!`.

For a successful GOUD delivery, chat normally contains only status, direct usable artifact locations or downloads, and one exact Mark decision request when necessary.
